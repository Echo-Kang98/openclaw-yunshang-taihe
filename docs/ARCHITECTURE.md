# 系统架构

## 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      用户层                              │
│         (飞书 / 企业微信 / 钉钉 / 网页)                    │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                    OpenClaw Gateway                      │
│                  (Agent 运行时引擎)                       │
└─────────────────────────────────────────────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│   三省       │   │   六部       │   │   钦天监     │
│  (协调层)    │   │  (执行层)    │   │  (监测层)    │
└──────────────┘   └──────────────┘   └──────────────┘
```

## Agent 体系

### 三省（协调层）

| Agent | 职责 |
|-------|------|
| 太子 | 皇上代理，总调度 |
| 中书省 | 方案起草 |
| 门下省 | 审核 |
| 尚书省 | 任务派发 |

### 六部（执行层）

| 部门 | Agent数 | 职责 |
|------|--------|------|
| 户部 | 1大+3小 | 财务经营 |
| 礼部 | 1大+3小 | 商务/行政 |
| 兵部 | 1大+3小 | 技术/运维 |
| 刑部 | 1大+3小 | 风控/合规 |
| 工部 | 1大+3小 | 开发/实施 |
| 吏部 | 1大+3小 | 人力资源 |

### 钦天监

| Agent | 职责 |
|-------|------|
| 钦天卫 | 外部信息监测、新闻简报 |

## Skills 分层

```
skills/              # 通用Skills（所有Agent可用）
├── self-improving/  # 自我提升
├── summarize/       # 内容摘要
├── notion/         # Notion集成
└── ...

skills-repo/dept-skills/  # 部门专属Skills
├── 高管层/
├── 财务经营部/
├── 国际交易部/
└── ...
```

## 数据流

1. 用户消息 → Channel (飞书/钉钉)
2. Gateway 路由 → 对应Agent
3. Agent 加载Skills → 执行任务
4. 结果返回 → 用户
