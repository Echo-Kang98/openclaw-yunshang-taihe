# 系统架构

## 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      用户层                              │
│         (飞书 / 企业微信 / 钉钉 / 网页)                    │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                    OpenClaw Gateway                      │
│                  (Agent 运行时引擎)                       │
└─────────────────────────────────────────────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│   高管层      │   │   六部       │   │   钦天卫     │

```

## Agent 体系

### 高管层（协调层）

| Agent | 职责 |
|-------|------|
| 太子 | 皇上代理，总调度 |
| 高管层 | 战略决策，会议纪要，任务分派 |

### 六部（执行层）

| 部门 | Agent数 | 职责 |
|------|--------|------|
| 财务经营部 | 1大+3小 | 财务报表、预算控制、资金管理 |
| 国际交易部 | 1大+3小 | 国际贸易执行、外汇对冲、合同管理 |
| 国内交易部 | 1大+3小 | 国内贸易执行、区域市场分析 |
| 交易执行部 | 1大+3小 | 订单执行、物流协调、船务管理 |
| 汇率经营部 | 1大+3小 | 汇率风险、对冲策略、套利监测 |
| 供应链运营部 | 1大+3小 | 供应链管理、库存优化、物流 |
| 期现交易部 | 1大+3小 | 期货交易、套利策略、期现对冲 |
| 卓越运营部 | 1大+3小 | 流程优化、KPI监控、效能分析 |
| 经营管理部 | 1大+3小 | 战略规划、绩效考核、经营分析 |

### 钦天卫

| Agent | 职责 |
|-------|------|
| 钦天卫 | 外部信息监测、新闻简报 |

```
├── 期现交易部/

```

## 数据流

1. 用户消息 → Channel (飞书/钉钉)
2. Gateway 路由 → 对应Agent
3. Agent 加载Skills → 执行任务
4. 结果返回 → 用户

## 泰和嘉柏业务背景

泰和嘉柏是一家从事**石油/大宗商品贸易**的企业，主要业务包括：
- 国际/国内原油及成品贸贸
- 期货交易与期现结合
- 外汇风险管理
- 供应链与物流管理
