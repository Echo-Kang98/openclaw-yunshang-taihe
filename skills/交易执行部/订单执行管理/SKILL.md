# 订单执行管理

## 功能定位

管理贸易订单从创建到关闭的全流程，确保订单按时、按质、按量完成，提升客户满意度。

## 订单生命周期

```
创建 → 确认 → 备货 → 发运 → 在途 → 到港 → 清关 → 交付 → 结算 → 关闭
```

### 1. 订单创建

**触发**：贸易合同签订

**录入信息**：
- 合同编号、签订日期
- 买方/卖方信息
- 货物品名、规格、数量、单价
- 装港、卸港、交货期
- 贸易条款（FOB/CIF/CFR）
- 付款方式（T/T LC D/P）

### 2. 订单确认

**确认事项**：
- 货物资源确认：库存/采购可行性
- 物流可行性：船期/舱位/运力
- 价格确认：与最新市场价格核对
- 客户需求确认：交期、品质要求

**输出**：订单确认书 → 进入执行阶段

### 3. 备货与发运

**备货管理**：
- 库存检查：现货是否充足
- 采购启动：库存不足时启动采购
- 品质检验：出库前品质确认
- 包装唛头：符合运输和客户要求

**发运安排**：
- 订舱/租船
- 物流运输至装港
- 装港报关
- 装货离港

### 4. 在途跟踪

**跟踪内容**：
- 航次/运单状态
- 当前位置（经纬度/港口）
- 预计到港时间
- 异常情况（如有）

**更新频率**：每日或关键节点更新

### 5. 到港与清关

**卸港操作**：
- 到港预报
- 卸港泊位安排
- 卸货计量（数量/品质）

**清关处理**：
- 进口报关
- 关税缴纳
- 商检查验（如有）
- 清关放行

### 6. 交付与结算

**交付确认**：
- 过磅/计量交接
- 品质验收
- 交付单据签收

**结算处理**：
- 发票开具
- 付款跟进（T/T 或 LC 议付）
- 核销归档

### 7. 订单关闭

**关闭条件**：
- 货物交付完成
- 全额收款
- 单证归档完毕

**关闭后处理**：
- 客户满意度回访
- 执行复盘
- 数据归档用于分析

## 订单状态

| 状态 | 说明 |
|------|------|
| 已创建 | 订单已录入系统 |
| 已确认 | 各项可行性确认完毕 |
| 备货中 | 货物准备中 |
| 已发运 | 货物已离港/离厂 |
| 在途中 | 运输过程中 |
| 已到港 | 抵达卸港 |
| 清关中 | 进口清关处理 |
| 已交付 | 客户已签收 |
| 已结算 | 款项已收付 |
| 已关闭 | 全流程完毕 |

## 异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 交期延误 | 提前预警，与客户协商 |
| 品质不符 | 协商降价/退货/索赔 |
| 数量短缺 | 补货或协商赔偿 |
| 客户拒收 | 协商解决方案，法律途径 |

## 回复模板

- 订单创建：`📝 订单{编号}已创建，卖方{卖方}，买方{买方}，金额{金额}`
- 状态更新：`📋 订单{编号}状态：{状态}`
- 发运通知：`🚢 订单{编号}已发运，航次{航次}，预计{日期}到港`
- 交付确认：`✅ 订单{编号}已交付，交付量{数量}，交付时间{时间}`
- 结算完成：`💰 订单{编号}已结算，结算金额{金额}，收款{日期}`
