# 流程优化诊断

## 功能定位

系统诊断核心业务流程，识别瓶颈和浪费，提出优化方案，推动流程再造，提升运营效率。

## 诊断范围

### 核心业务流程

| 流程 | 起点 | 终点 | 涉及部门 |
|------|------|------|----------|
| 贸易执行流程 | 合同签订 | 订单关闭 | 贸易部、风控、财务 |
| 采购执行流程 | 采购申请 | 验收入库 | 采购、仓储、财务 |
| 物流执行流程 | 发货指令 | 客户签收 | 物流、仓储、贸易 |
| 结算流程 | 发票开具 | 款项收付 | 财务、销售 |

## 诊断方法

### 1. 现状梳理

**流程图绘制**：
- 用流程图工具绘制现状
- 标注各环节：时长、操作人、系统
- 标识关键决策点

**数据收集**：
- 各环节平均时长
- 各环节异常率
- 各环节操作人数
- 系统使用情况

### 2. 瓶颈识别

**瓶颈特征**：
- 等待时间长（等待上游最长）
- 返工率高（重复操作多）
- 错误率高（差错多）
- 审批层级多（超3层审批）

**识别方法**：
- 帕累托分析：80%问题来自20%环节
- 时长分析：哪个环节耗时最长
- 异常分析：哪个环节问题最多

### 3. 根因分析

**5Why 分析法**：
```
问题：订单处理周期长
Why1：客户等待时间长 → 为什么？
Why2：审批环节多 → 为什么？
Why3：授权不充分 → 为什么？
Why4：担心风险 → 为什么？
Why5：缺乏数据支撑决策 → 根本原因
```

### 4. 优化方案

**优化方向**：

| 方向 | 说明 | 适用场景 |
|------|------|----------|
| 简化 | 减少环节/步骤 | 环节过多 |
| 自动化 | 系统替代人工 | 规则明确 |
| 并行化 | 多步骤同时执行 | 顺序依赖弱 |
| 标准化 | 统一操作规范 | 标准不一致 |

**方案评估**：
- 效率提升：预计缩短时长 {x}%
- 成本节省：预计年节省 {金额}
- 实施难度：低/中/高
- 投资回报期：{月}个月

## 优化案例参考

### 案例1：订单审批流程优化

**优化前**：5层审批，平均4.5天

**优化后**：
- 金额<10万：1层审批（部门负责人）
- 金额10-50万：2层审批
- 金额>50万：3层审批

**结果**：平均审批时长 4.5天 → 1.8天

### 案例2：入库流程自动化

**优化前**：人工录入系统，2人×4小时/天

**优化后**：地磅数据自动采集，系统自动入库

**结果**：人工减少 75%，错误率降至 0

## 回复模板

- 诊断完成：`🔍 {流程}诊断完成，发现{问题数}个瓶颈，最大瓶颈：{瓶颈}`
- 时长分析：`⏱ {流程}各环节时长：{环节1}{时长1} → {环节2}{时长2} → ... → 总时长{总时长}`
- 瓶颈定位：`⚠️ 瓶颈环节：{环节}，耗时{时长}，占比{占比}%，原因：{原因}`
- 优化建议：`💡 优化方案：{方案}，预计提升{提升}%，实施周期{周期}`
- 收益测算：`📊 优化收益：效率提升{效率}% + 成本节省{金额}/年，投资回报期{周期}`
