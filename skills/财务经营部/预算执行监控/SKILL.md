# 预算执行监控

## 功能定位

跟踪年度预算执行情况，监控预算收支进度，分析偏差原因，推动预算管控，确保年度预算目标达成。

## 预算体系

### 预算类型

| 类型 | 内容 | 编制时间 |
|------|------|----------|
| 收入预算 | 销售收入、其他收入 | 10-11月 |
| 成本预算 | 采购成本、运营成本 | 10-11月 |
| 费用预算 | 销售费用、管理费用、财务费用 | 10-11月 |
| 资本预算 | 固定资产投资 | 10-11月 |
| 资金预算 | 资金收支、融资 | 10-11月 |

### 预算分解

**纵向分解**：
- 集团预算 → 子公司预算 → 部门预算

**横向分解**：
- 年度预算 → 季度预算 → 月度预算

## 执行监控

### 月度跟踪

**跟踪指标**：
| 指标 | 年度预算 | 本月实际 | 累计实际 | 完成率 |
|------|----------|----------|----------|--------|
| 营业收入 | | | | |
| 毛利 | | | | |
| 净利润 | | | | |
| 销售费用 | | | | |
| 管理费用 | | | | |

### 执行率计算

```
月度执行率 = 本月实际 / 本月预算 × 100%
累计执行率 = 累计实际 / 累计预算 × 100%

偏差率 = (累计实际 - 累计预算) / 累计预算 × 100%
```

### 偏差分析

**偏差判断标准**：
| 偏差率 | 判断 | 行动 |
|--------|------|------|
| ±5%以内 | 正常 | 无需特殊行动 |
| 5-15% | 关注 | 分析原因 |
| >15% | 异常 | 预警并整改 |

**分析框架**：
```
收入偏差原因：
1. 销量变化 → 影响金额
2. 价格变化 → 影响金额
3. 结构变化 → 影响金额

费用偏差原因：
1. 人员变化 → 新增/减少
2. 政策变化 → 税率、费率调整
3. 管控不力 → 超标准支出
```

## 预警机制

### 预警等级

| 等级 | 条件 | 行动 |
|------|------|------|
| 🟢 正常 | 执行率 80-110% | 月度跟踪 |
| 🟡 关注 | 执行率 110-130% | 月度分析报告 |
| 🟠 预警 | 执行率 130-150% | 专项分析、审批收紧 |
| 🔴 严重 | 执行率 >150% | 立即整改、高管介入 |

### 预警流程

```
预警触发 → 通知责任部门 → 部门分析原因 → 制定整改措施
→ 财务审核 → 执行整改 → 跟踪效果 → 预警关闭
```

## 预算调整

### 调整原则

1. **刚性原则**：预算一经批准，原则上不调整
2. **例外原则**：重大市场变化或战略调整可申请调增
3. **规范原则**：必须履行规范的审批程序

### 调整流程

```
1. 部门提交预算调整申请
2. 说明调整原因
3. 分析调整影响
4. 财务审核
5. 预算委员会审批
6. 调整批准后执行
```

### 调增 vs 调减

| 类型 | 审批层级 | 说明 |
|------|----------|------|
| 收入调增 | 预算委员会 | 需说明市场机会 |
| 成本调增 | 预算委员会 | 需说明合理性 |
| 成本调减 | 财务部 | 需有替代方案 |

## 报告机制

### 月度预算执行报告

```
# XXXX年XX月预算执行报告

一、总体执行情况
二、各预算项执行详情
三、偏差分析
四、整改措施
五、下月预测
```

### 年度预算执行报告

- 全年预算执行总结
- 执行偏差分析
- 预算管理评价
- 下年度预算建议

## 回复模板

- 执行概览：`📉 {期间}预算执行：收入{收入}/{预算}万，完成率{率}%，费用{费用}/{预算}万，执行率{率}%`
- 偏差预警：`⚠️ 偏差预警：{部门}{预算项}偏差{偏差}万（{偏差率}%），{原因}，建议{措施}`
- 整改通知：`📋 预算整改通知：{部门}需对{事项}进行整改，期限{时间}`
- 调整申请：`📝 预算调整申请：{项}从{原预算}调至{新预算}，原因：{原因}，请{审批}`
- 调整批复：`✅ 预算调整批复：{项}调整已批准，{金额}万`
- 年度总结：`📊 年度预算执行总结：收入{收入}万，完成{率}%，费用{费用}万，执行{率}%，{评价}`
